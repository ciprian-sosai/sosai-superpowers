# Problem Framing

A technique for stating a problem clearly before investigating it. A poorly framed problem leads to solving the wrong thing. Used before Phase 1 of systematic-problem-solving.

## Why This Matters

"The report is bad" is not a problem statement — it's a complaint. Before you can find a root cause, you need to know precisely what is wrong, how you know it's wrong, and what "correct" would look like.

## The Problem Statement Template

```
**What is happening:** [Specific, observable description of the bad outcome]
**What should be happening:** [What correct looks like]
**How often:** [Always / sometimes / under specific conditions]
**Since when:** [First observed, or always been this way]
**Who is affected:** [Which stakeholders, which processes]
**Evidence:** [Specific examples, data, or observations]
```

**Example:**
```
What is happening: Monthly reports are missing the regional breakdown in section 3
What should be happening: Section 3 should show data split by North, South, East, West regions
How often: Every report for the past 3 months
Since when: Started after the Q4 2025 restructuring
Who is affected: Regional managers can't use the report for their team reviews
Evidence: Reports from Jan, Feb, Mar 2026 all show "National Total" only in section 3
```

## Common Framing Mistakes

| Mistake | Why it fails | Fix |
|---|---|---|
| "It's broken" | No specifics — can't investigate | State exactly what is wrong |
| "It's always been a problem" | No baseline — can't find what changed | Find a time when it worked or confirm it never did |
| "Stakeholders are unhappy" | Not a problem description — it's a reaction | Ask what specifically they expected vs. got |
| "The quality is low" | Unmeasurable — "quality" means different things | State which specific quality criteria are not met |
| Stating the solution as the problem | Closes off alternatives | "We need a new tool" should be "Results X are wrong because Y" |

## The Problem ≠ The Solution

A well-framed problem describes the gap, not the remedy.

| ❌ Solution-as-problem | ✅ Actual problem |
|---|---|
| "We need a better approval process" | "Approvals are taking 3 weeks when the target is 3 days" |
| "We need clearer briefs" | "Delivered work misses requirements in 60% of projects" |
| "We need more communication" | "Team learns of deadline changes after work is already done" |

State the gap. The solution comes after investigation — not before.

## Confirm Shared Understanding

Before investigating, confirm everyone agrees on the problem statement:
- Show your problem framing to the person who reported the issue
- Ask: "Is this what you mean?"
- Disagreement at this stage is valuable — it saves you from investigating the wrong thing
